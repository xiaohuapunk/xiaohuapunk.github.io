使用Hexo写博客

本文介绍了Hexo安装教程及简单使用方法

Hexo简介

  • Hexo 是一个轻量的静态博客框架。通过Hexo可以快速生成一个静态博客框架,仅需要几条命令就可以完成,相当方便。可以将Hexo的网站部署到互联网上免费的页面托管服务上，比如Github，Gitcafe,OSChian等等！

首先安装Nodejs

  • 地址：https://nodejs.org/

  • 点击页面中央绿色按钮下载安装包，按引导默认安装即可。

新建Github仓库

  • 教程（会的可以直接略过）

添加SSH-Key

  • 创建SSH key的方法很简单，在终端中执行如下命令就可以：

    ssh-keygen

  • 然后系统提示输入文件保存位置等信息，连续敲三次回车即可，生成的SSH key文件保存在

    ～/.ssh/id_rsa.pub

  • 然后用文本编辑工具打开该文件，复制所有内容，打开Github个人设置页面，选择SSH keys - 点击Add SSH key粘贴即可。

开始安装Hexo

  • 安装命令

    npm install -g hexo

  • 若安装失败，尝试

    sudo npm install -g hexo

  • 初始化hexo文件夹

    /*
 * init命令后跟指定目录 或 cd到指定目录直接hexo init
 */
hexo init

  • 生产静态页面并预览 执行完下面的命令后在浏览器中输入 localhost:4000就可以访问Hexo站点了

    hexo g && hexo s

目录结构

  • Hexo的目录结构如下，其中_config.yml是整个站点的配置文件，source/_drafts是草稿文件目录，source/_posts是发表的博文目录，themes是下载的主题目录，其它的目录暂时不用管。
.
├── _config.yml
├── package.json
├── scaffolds
├── scripts
|   ├── _drafts
|   └── _posts
└── themes

  • 修改配置文件，主要就是配置一下自己的站点信息，配置文件中的注释比较详细，如果不懂的话可以查看说明。

  • 修改主题，基本就是git clone到themes目录下然后修改_config.yml中的theme字段就可以了，十分方便，主题下载。

写文章

  • 新建文章

    hexo new "Title"

  • 格式说明(Markdown 方式编辑)

    title: postName  /* 标题 */
date: 2013-12-02 15:30:16  /* 时间，会自动生成 */
categories: TEXT  /* 分类 页面右侧导航 */
tags: BLOG  /* 标签 页面右侧导航 */
---

文章摘要，当在首页显示时，会增加read more 按钮
<!--more-->      /* 文章摘要分割线 */
正文 

*********************************************************************        
layout: false  /* 如果不需要hexo负责布局 在文章开头添加 */

  • 新建菜单

    hexo new page "about"

  • 将新建菜单类型配置到页面

    打开主题文件夹 ~/HEXO/themes/light/_config.yml
记事本打开_config.yml文件
menu:        
      Home: /
      Library: /archives
      About: /about

  • 部署到Git前的配置修改

    /* 编辑hexo主目录下的_config.yml文件，账号换为自己的Github账号 */
# Deployment
## Docs: http://hexo.io/docs/deployment.html
deploy:
      type: git
      repository: https://github.com/offfield/offfield.github.io.git
       branch: master

/* 若失败，请尝试 */
npm install hexo-deployer-git --save

  • 部署到Github

    hexo deploy
/* 查看结果访问：https://(username).github.io  */

  • 常用命令

    hexo n = hexo new
hexo g = hexo generate
hexo s = hexo server
hexo d = hexo deploy

  • 注意事项

    Hexo命令必须在init目录下执行;
每次修改如果显示不成功，先执行hexo clean 再 hexo generate;
每条带冒号命令后面必须有一个空格，否则会报错;
